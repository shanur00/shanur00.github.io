Flutter Notes: IconButton Styling

1. Selection (Toggling States)

An IconButton can act as a toggle switch. This requires managing state to flip between the default icon and the selected icon.

bool isFavorited = false;

// Inside the build method:
IconButton(
  iconSize: 72,
  icon: const Icon(Icons.favorite_border),
  isSelected: isFavorited,
  selectedIcon: const Icon(Icons.favorite, color: Colors.amber),
  onPressed: () {
    setState(() {
      isFavorited = !isFavorited;
    });
  },
)
Important Rules:
  • The isSelected property determines which icon is displayed.
  • When isSelected is false, the widget assigned to icon is shown.
  • When isSelected is true, the widget assigned to selectedIcon is shown.
  • You must wrap the boolean toggle inside setState(() { ... }) within the onPressed callback to force the UI to rebuild and show the new state.

2. IconTheme vs IconButtonTheme

Flutter resolves the styles for an IconButton based on a strict order of precedence. A more specific theme or a direct widget property will override general themes.

// Inside MaterialApp:
theme: ThemeData(
  iconTheme: const IconThemeData(color: Colors.green),
  iconButtonTheme: IconButtonThemeData(
    style: IconButton.styleFrom(
      foregroundColor: Colors.blue,
    ),
  ),
),

// Inside the Widget Tree:
IconButton(
  color: Colors.red,
  // ... other properties
)
Important Rules:
  • Priority 1 (Widget Property): The color: Colors.red applied directly to the IconButton is the strongest and overrides all themes.
  • Priority 2 (IconButtonTheme): If the direct color property was missing, it would fallback to Colors.blue. The IconButtonTheme overrides the general IconTheme because it is a widget-specific theme.
  • Priority 3 (IconTheme): The Colors.green acts as the absolute baseline. It only applies if the widget is a standalone Icon and has no specific widget theme or direct color property applied to it.

3. Visual Density (Shape & Size)

Visual density adjusts the compactness of the button. In Material Design 3, altering the visual density changes the background shape of the IconButton. It can be set at the theme level or on the widget directly.

// 1. Inside MaterialApp Theme:
theme: ThemeData(
  iconButtonTheme: IconButtonThemeData(
    style: IconButton.styleFrom(
      visualDensity: VisualDensity.compact,
    ),
  ),
),

// 2. Inside the Widget Tree:
IconButton(
  style: IconButton.styleFrom(visualDensity: VisualDensity.standard),
  visualDensity: VisualDensity.compact,
)
Important Rules:
  • VisualDensity.standard is the default in Material 3 and gives the IconButton a perfect circular shape.
  • VisualDensity.compact reduces the padding and transforms the shape into a rounded rectangle.
  • Theme Level vs Widget Level: The compact setting in the ThemeData is applied to all IconButtons globally, unless a specific button overrides it.
  • Conflict Resolution (Widget overrides Theme): Even though the global theme sets density to compact, the button's explicit style property overrides the theme. In this code, the button remains circular because the style property enforces standard, overriding both the theme and the direct visualDensity shortcut property below it.

4. Full Source Code

The complete main.dart implementation demonstrating these rules in action.

import 'package:flutter/material.dart';

void main() {
  runApp(
    MaterialApp(
      theme: ThemeData(
        //[IconTheme] PRIORITY 4: IconTheme (Weakest overall)
        // General fallback for all standalone icons.
        iconTheme: const IconThemeData(color: Colors.green),
        
        //[IconTheme VS IconButtonTheme] PRIORITY 3: IconButtonTheme
        // This beats the general IconTheme because it is specifically for IconButtons.
        iconButtonTheme: IconButtonThemeData(
          style: IconButton.styleFrom(
            foregroundColor: Colors.blue,
            visualDensity: VisualDensity.compact,
          ),
        ),
      ),
      home: const MainApp(),
    ),
  );
}

class MainApp extends StatefulWidget {
  const MainApp({super.key});

  @override
  State createState() {
    return _MainAppState();
  }
}

class _MainAppState extends State {
  bool isFavorited = false;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Center(
        child: IconButton(
          //[Widget Property] PRIORITY 2: Direct Widget Property (Shortcut)
          // This will color the icon red, overriding the themes above.
          color: Colors.red,
          
          //[styleFrom] PRIORITY 1: Widget Style (The Ultimate Boss - Strongest)
          // This overrides EVERYTHING else. Since we set the visualDensity to standard here,
          // it takes absolute control of the button's shape.
          style: IconButton.styleFrom(visualDensity: VisualDensity.standard),
          
          iconSize: 72,
          icon: const Icon(Icons.favorite_border),
          isSelected: isFavorited,
          selectedIcon: const Icon(Icons.favorite, color: Colors.amber),
          
          //[visualDensity] PRIORITY 2: Direct Widget Property (Shortcut)
          // WHY THIS HAS NO EFFECT: Because the Ultimate Boss (`style` property above) 
          // is already actively setting the visualDensity. The Boss ignores this shortcut.
          visualDensity: VisualDensity.compact, 
          
          onPressed: () {
            setState(() {
              isFavorited = !isFavorited;
            });
          },
        ),
      ),
    );
  }
}
